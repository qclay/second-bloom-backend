# Push-уведомления (Firebase FCM)

## Приём FCM-токена

**Эндпоинт:** `POST /api/v1/users/fcm-token`  
**Авторизация:** Bearer JWT

**Тело запроса:**
```json
{
  "fcmToken": "fGhJkLmNoPqRsTuVwXyZ1234567890..."
}
```

Чтобы сбросить токен (например, при выходе):
```json
{
  "fcmToken": null
}
```
или
```json
{
  "fcmToken": ""
}
```

**Ответ:** обновлённый профиль пользователя (200 OK).

---

## Когда вызывать на мобилке (Flutter)

1. **После успешной аутентификации** — получите FCM токен через Firebase Messaging и вызовите `POST /users/fcm-token` с этим токеном.
2. **При каждом запуске приложения** (если пользователь залогинен) — токен может обновляться, лучше отправлять актуальный.
3. **При выходе из аккаунта** — вызовите с `fcmToken: null` или пустой строкой, чтобы не получать уведомления после logout.

## Триггеры push-уведомлений

Бэкенд отправляет push в следующих случаях (при наличии у пользователя FCM токена и включённых настройках в NotificationPreference):

| Событие | Получатель | Описание |
|---------|------------|----------|
| **NEW_MESSAGE** | Покупатель/продавец | Новое сообщение в чате |
| **NEW_BID** | Продавец | Новая ставка на его аукцион |
| **OUTBID** | Покупатель | Его ставку перебили |
| **BID_REJECTED** | Покупатель | Ставка отклонена продавцом |
| **AUCTION_ENDED** | Участник аукциона | Аукцион завершён (с указанием, выиграл ли) |
| **ORDER_CONFIRMED** | Покупатель | Продавец подтвердил заказ |
| **ORDER_SHIPPED** | Покупатель | Продавец отметил заказ как отправленный |
| **ORDER_DELIVERED** | Покупатель | Продавец отметил заказ как доставленный |

## Структура data в push

В payload push-уведомления передаётся `data` (для навигации во Flutter):

```json
{
  "notificationId": "uuid",
  "type": "NEW_MESSAGE",
  "conversationId": "uuid",
  "productId": "uuid",
  "orderId": "uuid",
  "auctionId": "uuid"
}
```

Поля зависят от типа уведомления. Используйте `type` для выбора экрана.
